Developer Architecture & API Guidelines 🛠️

Complete technical specifications for Flutter client compilation, database schemes, and email triggers.

This developer portal details the serverless architecture of BabyShopHub. The project utilizes a decoupled approach: a cross-platform Flutter Mobile Client that connects directly to the Firebase Emulator Suite locally, and triggers Node.js Cloud Functions to handle transactional email dispatches via Gmail SMTP.

Directory & Project Layout 📂

The codebase is organized into key modules to allow clean maintenance and robust testing:

babyshophub/
├── firebase.json         # Local Emulator port mapping configuration
├── firestore.rules       # Security and query path assertions
├── firebase/
│   └── functions/        # Node.js Background triggers package
│       ├── index.js      # Gmail SMTP trigger routines (Nodemailer)
│       └── package.json  # Functions dependencies
├── docs/                 # Documentation folders
│   ├── user/             # Parent Help Portal

    ├── theme/            # Theme Engine & SharedPrefs states
    ├── models/           # Dart Object classes (Cart, User, Products)
    └── screens/          # Mobile View controllers

Setting up Firebase Local Emulators 🚀

To run, test, and debug auth systems, databases, and emails locally without cloud billing, follow these steps:

Pre-requisite: Ensure Node.js and the Firebase CLI tools are installed: npm install -g firebase-tools

  1. Open a terminal in the root workspace directory.
  2. Install the Cloud Functions package dependencies: 
    cd firebase/functions && npm install
  3. Configure SMTP credentials by copying .env.example to .env inside the functions directory and populating it with your Google App Passwords.
  4. Start the emulator suite: 
    firebase emulators:start
  5. Launch the Flutter Web app pointing to localhost emulators: 
    flutter run -d chrome --web-port=5000

Cloud Functions & SMTP Emails 📧

Background events listen to Firestore mutations to trigger Nodemailer dispatches. The triggers are:

  • onUserCreated (users/{uid}): Generates a welcome email containing instructions.
  • onOrderCreated (orders/{orderId}): Renders a dynamic receipt HTML table and emails it instantly.
  • onProductStockUpdated (products/{productId}): Listens to stock modifications. If stock transitions from 0 to >0, it queries all users who have this productId in their active wishlist, gathers their addresses, and fires back-in-stock alerts.
  • onLoginLocationCheck (logins/{loginId}): Detects login locations. If simulated threat IP checks fail, dispatches a security login warning alert.

Dynamic Theme Provider & Styles 🎨

The client application contains a robust ThemeProvider extending ChangeNotifier. It allows picking theme styles preset packages or custom RGB hexadecimal codes (like #FF9EAA) for primary/secondary configurations.

Color states are automatically serialized and saved locally to disk using SharedPreferences on changes. On boot-up, the app reads these keys and loads the customized themes, preventing styling lag.
